# 区域赛奖励诊断实例

## 实例1：智能体乱放技能

**现象**：智能体CD好了就释放技能，不瞄准敌方英雄

**诊断过程**：
1. 读 conf.py SKILL_HIT_WEIGHT：鲁班s1=0.008, s2=0.01, s3=0.0
2. 对比其他奖励：hp_point=2.0, last_hit=0.5
3. 计算：一次技能命中 +0.008，补刀一个兵约 +0.5×差分
4. 结论：技能命中奖励是补刀的 1/60，几乎无感

**根因**：技能命中权重过低(0.008)，无浪费/落空惩罚

**修复**：
- SKILL_HIT_WEIGHT 提升 5-6 倍（0.008→0.045）
- 新增技能落空惩罚（释放但未命中 → -0.005）

## 实例2：智能体徘徊不前

**现象**：智能体在兵线前后摇摆，不坚定推进

**诊断过程**：
1. 读 conf.py：forward=0.02, death_point=-1.0
2. 分析奖励信号：forward把智能体往前推，death把它往回拉
3. 检查战斗能力：skill_hit权重极低 → 打不赢 → 死亡 → 恐惧
4. 结论：forward-death振荡

**根因**：战斗能力不足（技能奖励弱），导致"冲上去→被打死→撤退→再冲"循环

**修复**：
- forward 权重从 0.02 降到 0.01（减少拉扯）
- 技能命中权重提升（增强战斗能力）
- kill_point 从 -0.2 改为 0.2（消除击杀惩罚）

## 实例3：智能体不打英雄

**现象**：智能体优先补刀小兵，回避与敌方英雄战斗

**诊断过程**：
1. 读 conf.py：kill_point=-0.2, last_hit=0.5
2. 计算：击杀英雄 reward = +1×(-0.2) = -0.2（扣分！）
3. 计算：补刀小兵 reward = +0.5×差分（正奖励）
4. 结论：打英雄亏本，补刀稳赚

**根因**：kill_point 为负值，直接惩罚击杀行为

**修复**：
- kill_point 从 -0.2 改为 0.2（击杀应该奖励）

## 实例4：value_loss_econ 异常高

**现象**：训练监控中 value_loss_econ=1.22，是其他头的10-25倍

**诊断过程**：
1. 读 conf.py：money_point=0.006, exp_point=0.0055
2. 分析数据特性：money_cnt 是累计值，差分方差极大
3. 值网络需要拟合一个方差极大但权重极小的信号
4. 结论：值网络在经济维度拟合失败

**根因**：money_point 权重太小(0.006)但数据方差极大，值网络无法有效拟合

**可能修复**：
- 适当提高 money_point 权重
- 或对经济数据做归一化/裁剪
